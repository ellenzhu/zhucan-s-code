Webpage - 3 Layers

There are three distinct qualities to a web page that you can observe: content, look, and behavior

In other words, what is on the page, how does it look, and if something happens how does the page react.

Style - Cascading Style Sheets

CSS enables the separation of concerns: content and presentation.

Affects the presentation of HTML and can be customized to different media as well as screen & print.

Cascading means one page may have (and often does) different styles, which have different priority. Highest priority (i.e. most specific definition) determines the presentation.

Advantages: separation of concerns, site-wide consistency, bandwidth, accessibility

Why The Separation?

Coupling

When style is separated from the content one can be modified without affecting the other one.

Most pages on a website look the same, this can be achieved by meticulously editing every page to have a same look... or by using (a single) stylesheet that is loaded on all the pages.

"The mark of maintainable HTML, CSS, and JavaScript is when individual developers can easily and confidently edit parts of the code base without those changes inadvertently affecting other, unrelated parts."

HTTP

Hypertext Transfer Protocol

HTTP - How Content Gets Around

HTTP stands for Hypertext Transfer Protocol.

It's an application level protocol (OSI 8) which is stateless.

It follows client-server model, which means that a client makes a request to the server and the server provides a response (which might also be an error).

Since it is stateless, no state is kept between two requests.

HTTP Example

A request to the server might look something like this:

GET / HTTP/1.1
Host: developer.mozilla.org
Accept-Language: fr

To which the server might respond with the following:

HTTP/1.1 200 OK
Date: Sat, 09 Oct 2010 14:28:02 GMT
Server: Apache
Last-Modified: Tue, 01 Dec 2009 20:18:22 GMT
ETag: "51142bc1-7449-479b075b2891b"
Accept-Ranges: bytes
Content-Length: 29769
Content-Type: text/html

<!DOCTYPE html... (here comes the 29769 bytes of the requested web page)

Example taken from MDN - HTTP. You can also inspect these requests on your browser.

HTTP - Safe and Idempotent Methods

There are safe methods GET and HEAD: these will fetch some data and importantly will not have any side-effects on the server.

Idempotent methods are methods that behave in a way where doing multiple identical requests will result in the same side-effects on a server as a single request. Safe methods are idempotent, but in addition there are PUT and DELETE

PUT uploads a resource to the server. If it does not exist it gets created. If it exists (and different from request) it gets updated.

DELETE removes a resource from the server.

HTTP - POST Method

As the name implies, the POST method will post some data to the server. It can have side effects on the server and it is up to you what you do with it. An example from MDN: 

POST /contact_form.php HTTP/1.1
Host: developer.mozilla.org
Content-Length: 64
Content-Type: application/x-www-form-urlencoded

name=Joe%20User&request=Send%20me%20one%20of%20your%20catalogue

There are also a few more METHODs, you can read more from the W3C specification.

HTTP - Response Codes

GitHub's take on 404 page

HTTP - Response Codes

HTTP Responses have a response code which is three numbers long:

  • 1xx Informational responses, e.g. 100 Continue
  • 2xx Success responses, most common 200 OK
  • 3xx Redirect responses, e.g. 301 Moved Permanently
  • 4xx Client error, you've all seen 404 Not found, 403 Forbidden is also common.
  • 5xx Server errors, like 500 Internal Server Error

There is also error 418 I'm a teapot where "The resulting entity body MAY be short and stout."

What content is on the page?

Content

The content is defined by something called HTML, which stands for HyperText Markup Language. It holds the words you are reading currently, it has the menu items on a website.

It's HyperText - it can point (commonly known as link) to other HyperText documents.

Check Wikipedia for the history of the word.

Markup means, that you construct it by marking up text in a certain manner.

Language comes from the fact that it has a specific way of expressing things.

HTML... the content stuff

So HTML is a language, it has syntax and a grammar. It is a way of expressing content.

If it is conscructed correctly we say it is valid.

Throughout the course we expect that you only submit valid HTML (and CSS). Checking validity of HTML/CSS can be done with W3C's validator service: http://validator.w3.org

Document structure

HTML documents start with a doctype definition, for HTML5 we just use <!DOCTYPE html>.

Rest of the document is then presented inside <html>-element in two major sections: <head> and <body>

Inside <head> document title and meta elements are set. Meta elements describe the document (who was the author, what character encoding is to be used, etc). Additionally, stylesheets and scripts can be loaded in here.

<body> contains the actual information in the document and is shown in the browser. Elements we've covered so far all go inside the <body>.

Character encoding

To explicitly state what character encoding is used on the page <meta charset="utf-8"> is inserted at the start of the <head> of the document.

Encodings are easy to get wrong and there are typically multiple point of failures when loading a web page (browser, the document, server or database might declare something wrong.

CSS

What does the page look like?

CSS

Cascading Style Sheets or CSS for short defines the outlook of HTML.

It can be set in three different ways:

  • Inside elements in the style-attribute
  • Inside <head> in <style>-element
  • In a separate file that's loaded inside <head> with <link> element.

Loading style sheets from separate file is preferred. One file can be used on multiple pages – meaning that changing the style will affect all of them.

CSS Examples

Inside style-attribute:

<p>Some text and <span style="color: blue;">some blue text</span></p>

In <style>-element:

<head>
  <title>A simple page</title>
  <style> h1 { border-bottom: 2px solid black; }</style>
</head>

From a separate file:

<html>
  <head>
  <title>A simple page</title>
  <link rel="stylesheet" href="my_awesome_style.css">
  </head>
  <body>
    <h1>Some content</h1>
  </body>
</html>

How CSS works

CSS is simply a list of style rules that are applied to particular elements

Rules consist of a selector and then one or more declarations

Changing style for all p elements:

p {
  font-family: serif;
  color: #0000FF;
  }

Changing style for all a-elements that have a class main_menu:

a.main_menu {
  font-size: 20px;
  padding: 10px;
  }
<a href="index.html" class="main_menu">Home page</a>
<a href="about.html" class="main_menu">About us</a>
<a href="other.html" class="other_link">Other Links</a>
<!-- last link will not be affected -->

Class, ID, and Element as selectors

As in previous slide, elements class-attribute is often used to select particular elements for styling. A single dot is placed before the class name to select it in CSS.

/* Make all paragraphs that have info_text as class have italic font-style */
p.info_text { font-style: italic; }

The id-attribute can also be used. Id must always be unique for the page. This way one particular element is selected for styling. Hash-character is used to denote id in CSS.

/* div with the id of notice will have red borders */
/* Note that the div is reduntant (and makes selecting it slower) */
div#notice { border: 1px solid red; }
/* Since at most, there will be exactly one element with the id of div
leaving div out of the selector is preferred. */
#notice { border: 1px solid red; }

A third option is to use element type as a selector:

/* Add underline to all h1 elements */
h1 { text-decoration: underline }

Classes as selectors

Class selectors can also be used without specifying any elements. In this case any element with that particular class is affected. So with the following rule:

.green_text {color: green}

...both of the following and any other elements with the class green text will be affected.


<h1 class="green_text">Ugly Heading</h1>
<p>Lorem ipsum <span class="green_text">sit dolor</span> amet.</p>

Multiple Selectors and Selecting Multiple Things

Multiple classes can also be required by concatenating classes:

p.old.python_code { text-decoration:line-through; }
<p class="old python_code">print "Hello world!"</p> <!-- affected -->
<p class="new python_code"> print("Hello world!")</p> <!-- not affected -->

Note the space between new/old and python_code. These paragraphs have two classes.

Multiple elements, classes and ids can be selected with comma

/* Affects all h1, h2, and h3 elements */
h1, h2, h3 { font-family: serif; }

Descendant, Child, and Sibling Selectors

Descendant selector selects all elements contained inside the parent element.

/* All li elements inside element with id of important */
#important li { font-weight: bold; }

Child selector selects all elements which are immediate children of the parent element.

/* All h2 elements that are immediate children of  element with id of content */
#content > h2 { font-weight: bold; }

Sibling selectors select any elements that share the same parent and are after the first selector.

/* Any p that share parent with h1 that are after h1 */
h1 ~ p { font-weight: bold; }

Next sibling selector selects the elements which is immediately after the defined sibling.

/* Selects p elements that are immediately after h1 */
h1 + p { font-weight: bold; }

Pseudo-elements

There are also pseudo-elements, which can be appended to a selector. They affect specific parts of the document. e.g. p::first-line will affect the first line of paragraphs.

In CSS3 (CSS has versions too) double colons are used to distinguish them from pseudo-classes (coming up next).

All of the pseudo-elements are:

  • ::after
  • ::before
  • ::first-letter
  • ::first-line
  • ::selection (non-standard)

Pseudo-classes

Pseudo-classes affect elements with a specific state. They are marked with single colon, e.g. :hover.

/* this one is applied to the link at the bottom of the slide */
a.special_link:hover { background-color: white; }

Some selected ones:

  • :active
  • :enabled
  • :first-child
  • :hover
  • :nth-child(an+b)
  • :visited

All pseudo-classes listed at MDN

CSS - Cascading

Cascading ...or what style gets applied if there's conflict?

First the rules which apply to that element are selected then they are ordered followingly :

  Origin Importance
1 user agent normal
2 user agent !important
3 user normal
4 author normal
5 CSS Animations (refer to links on next slide)
6 author !important
7 user !important

This has the very important implication that if I, as a user, decide to override all font-families with comic-sans and note it as !important then all of your hard work selecting fonts will be in vain!

CSS - Cascading - Specificity

It is still possible (and even likely) that multiple styles still apply. The result will finally come from specificity rules.

  • Universal selectors noted as explicitly as *, but can be omitted
  • Type selectors
  • Class selectors
  • Attributes selectors (they check wheter attribute is present or has particular value)
  • Pseudo-classes
  • ID selectors
  • Inline style

!important command is used to override the cascading properties of CSS. You should only use it when there is no other way! In practise, when there is CSS from somewhere that you can't control.

In case there is equality, the latest declaration in the CSS will be applied.

This and previous slide based heavily on MDN Cascading and Specificity articles, do check them out.

CSS - Fonts - Font-family

font-family declarations control what fonts are used for the selector. It's value is comma separated list of fonts (a font stack) that should have a generic family as the last item.

If the first font on the list is not found the next one is tried until a font is found. The generic families for the fonts are: serif, sans-serif, monospace, cursive, and fantasy.

p.ingress {
  font-family: "Gill Sans", Georgia, "Times New Roman", serif;
}

CSS - Margin, Padding, & Border

Margins are always transparent. Horizontal margins don't overlap (collapse) while vertical (top and bottom) margins can collapse.

h2 { margin-bottom: 10px; }
p { margin-top: 5px;}
<h2>Collapsing Margins</h2>
<p>The margin between this paragraphs and
the heading above would be 10 pixels</p>

Padding is the area surrounding the content box that shares its background.

Border is between padding and margin and has: color, width, and style:

/* These two are equivalent */
h2 { border: 3px solid #FF0000; }
h2 { border-width: 3px; border-style: solid; border-color: #FF0000; }

CSS - Inline & Block

Elements are divided into inline and block elements. Inline elements, such as span and a flow "like text" from left to right without line breaks.

Block elements (such as div and p) inhabit the entire space of parent element (remember box model from couple slides ago?). Typically there is a new line inserted before and after them.

Elements can be changed from inline to block by display declarations.

/* Would insert line breaks before and 
after links with the class menu_item */
a.menu_item { display: block; }

CSS - Position

The position of an element can be set with position-attribute

It can be set as:

  • static The element is positioned as in normal flow
  • relative Element is positioned relative to it's "normal" position
  • absolute Element is positioned relative to nearest positioned ancestor or if there is no positioned ancestor to the container of the document
  • fixed Element is position relative to the viewport. It has the effect of keeping the element at the same position in the window even when scrolling.
  • sticky Mixture of relative and fixed positioning. Not yet widely supported.

MDN on Position

CSS - Position Example

/* Will always stay at the top left corner of the viewport */

  position: fixed;
  top: 20px;
  left: 20px;
}
/* Will be 20 pixels lower and 20 pixels to 
the left of where it would be without this style */
p.side_note {
  position: relative;
  top: 20px;
  left: 20px;
}
