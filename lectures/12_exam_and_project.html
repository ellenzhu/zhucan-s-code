Project

Project Topic

The topic for the course project for CSE-C3210 2014-2015 is an online game store for JavaScript games. The service has two types of users: players and developers. Developers can add their games to the service and set a price for it. Players can buy games on the platform and then play purchased games online.

You should carefully read through the full project description.

Important Dates

Deadlines

  • Group registration - 13.12.2014 midnight
  • Project Plan - 20.12.2014 midnight
  • Final submission - 13.2.2015 midnight (end of period III)

The project is completed in groups of three and it is done with Django. jQuery is the recommended JavaScript library, but you are free to use other ones as well.

Outline for The Project

  1. You form a group of 3 and create a project in GitLab. Assign wsd-agent as a reporter.
  2. You write a project plan. After the plan is finished, create an issue in GitLab and assign it to wsd-agent.
  3. We'll review and comment your project plan.
  4. You start implementing the project. Once you are done, deploy it to Heroku (at least one from your group needs to register).
  5. Once you are done, you'll write final submission and create a new issue informing that you are ready for grading.
  6. Time for grading will be arranged. Grading is a 30 minute demonstration of your project where all group members need to be present.

Practical Tips

Agree upon working habits and choose a way to collaborate. There are tools out there to facilitate team work (e.g. Trello, Flowdock, Google Drive). Long and confusing email chains will just hurt you.

Remember planning, even after the initial project plan. Some important things to plan ahead are URL structure and models that you will use.

First build a functional application (minimal working implementation), and only after that start polishing and adding 'nice-to-have' features.
