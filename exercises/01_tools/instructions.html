2. Getting Started With Git

Description

This exercise is meant as a gentle introduction to the git workflow used during this course. You will learn how to clone your git project to a computer, edit a file, add those edits, commit your changes and finally push them to your repository at GitLab. We check the exercises/01_tools/hello.html in your GitLab repository and expect to see the following changes (editing the file is easiest part of this exercise):

  • Document hello.html will be valid html5
  • Content of the meta element with the name of author will be changed
  • Content of em with the id of signature will be changed
  • The changed meta and #id will be the same

Short Instructions

  1. Install git (if not already installed) to your computer
  2. Generate and set your SSH key to GitLab
  3. Clone your wsd-material project to your local computer
  4. Solve the exercise as detailed in the description
  5. Add your changes, commit them and then push your commit to GitLab
  6. Submit Exercise to A+

Detailed instructions

1. Installing git

If you work from Paniikki you can skip to the next section, git is already installed. You can also test if git is installed by just typing git in terminal.

If you do not have git already installed on your computer, where you intend to work, you can download git for various platforms from the official site: http://git-scm.com/downloads. The Windows installation package includes Git Bash that you should use to run the terminal commands given through out the course instructions.

There are also graphical tools for git, such as EGit for Eclipse that you can use but no support for these are provided.

2. Generate and set SSH key

To enable cloning your project the your local computer you need to add a SSH key to your GitLab profile.

Follow the instructions on how to generate a SSH key and then add that key to GitLab (found at https://git.niksula.hut.fi/profile)

$ ssh-keygen -t rsa
# (you should type enter to all questions)
$ cat .ssh/rsa_id.pub
# (use window menu or right click to mark and copy the key in Git Bash)

3. Cloning a project

Now you are ready to clone your project on your local machine. Choose a directory and type:
git clone git@git.niksula.hut.fi:YOUR_USER_NAME/wsd-material.git
this will authenticate you on the server with the SSH key you created in the previous step and create a folder with the name of wsd-material where the contents of your repository from your server are cloned. If you did not change directories in Git Bash look for C:\Users\<username>\wsd-material. 

    $ git clone git@git.niksula.hut.fi:YOUR_USER_NAME/wsd-material.git
    
    Cloning into 'wsd-material'...
    remote: Counting objects: 3, done.
    remote: Total 3 (delta 0), reused 0 (delta 0)
    Receiving objects: 100% (3/3), done.
    Checking connectivity... done.

Note that GitLab also has a feature where you can download your project. If you download your project instead of cloning you will not be able to submit your exercises.

4. Solving The Exercise

Open up exercises/01_tools/hello.html on your favorite code editor... or if you don't have a code editor read review of editors that are particularly well suited for this course.

You are free to edit the hello.html file to test things out, it just needs to match following four criteria so you get the points from this exercise:

  • Document hello.html will be valid html5
  • Content of the meta element with the name of author will be changed
  • Content of em with the id of signature will be changed
  • The changed meta and #id will be the same

5. Add, Commit and Push changes

After you've modified the hello.html file, you need to commit your modifications. First check the status of the project:

    $ git status

    On branch master
    Your branch is up-to-date with 'origin/master'.

    Changes not staged for commit:
      (use "git add <file>..." to update what will be committed)
      (use "git checkout -- <file>..." to discard changes in working directory)

            modified:   exercises/01_tools/hello.html

    no changes added to commit (use "git add" and/or "git commit -a")

To stage files for a commit you first need to add them. Note that add does not refer to adding a new file to your project but adding a file to a commit.

    $ git add exercises/01_tools/hello.html
Again you can check with git status that the file will be included in the commit: 
    $ git status

    On branch master
    Your branch is up-to-date with 'origin/master'.

    Changes to be committed:
      (use "git reset HEAD <file>..." to unstage)

          modified:   exercises/01_tools/hello.html

Now you can commit these changes to your repository with git commit. It will open a text editor (on linux the default is typically vim) where you can write a message for the commit that reflects the changes that you've done. You can also just type the commit message alongside the command with -m switch, for example:

    $ git commit -m "Solved the first exercise"
    [master ab66b8d] Solved the first exercise
    1 file changed, 2 insertions(+), 2 deletion(-)

Now the commit is done and the changes you've made to the files that you committed are tracked in the repository history. You can verify this by command: git log.The first entry should look something like this:

    $ git log
    commit 2b9c171c4f051cd46b5ce3d77141aaef5bfdce9a
    Author: Firstname Lastname <YOUR_USER_NAME@niksula.hut.fi>
    Date:   Sun Oct 12 19:01:29 2014 +0300

    Solved the first exercise

What remains is pushing these changes to your repository in GitLab. This can be done with: git push: 

    $ git push
    Counting objects: 7, done.
    Delta compression using up to 8 threads.
    Compressing objects: 100% (3/3), done.
    Writing objects: 100% (4/4), 345 bytes, done.
    Total 4 (delta 2), reused 0 (delta 0)
    To git@git.niksula.hut.fi:xxxxxxxx/wsd-material.git
    eb2fdc8..ab66b8d  master -> master

If there were no errors the changes are now reflected in GitLab (you can browse your project's files if you want to verify this).
